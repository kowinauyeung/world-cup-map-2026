# Project documentation

| Area | Purpose |
| --- | --- |
| [`requirements/`](./requirements/) | Testable product behaviour and the source-data contract. |
| [`decisions/`](./decisions/) | Architecture Decision Records (ADRs) that constrain the implementation. |
| [`implementation/`](./implementation/) | Component boundaries, delivery order, and test strategy for implementation. |

Implementation must satisfy the requirements and follow accepted ADRs. If a requirement or ADR needs to change, update the relevant document in the same change and state why in the pull request.
